# Design: рестайл матрицы «Каталог стандартов» в языке ReviewItemsTable

**Дата:** 2026-07-15
**Статус:** черновик на ревью
**Автор:** агент + заказчик (brainstorming)
**Мокап:** `scratchpad/matrix-restyle-mockup.html` (артефакт, обе темы)

## Контекст и мотивация

Заказчику понравилась таблица `ReviewItemsTable` из соседнего проекта UDP
(`frontend/src/components/review/ReviewItemsTable.tsx`) — плотная grid-таблица с
sunken-шапкой, чип-языком ячеек и переключением view↔edit через `<fieldset>`.
Возник вопрос: организовать «Каталог стандартов» так же.

**Разведка (см. ход брейншторма) дала три вывода:**

1. `ReviewItemsTable` — это **плоский список** одной сущности с ФИКСИРОВАННЫМ
   набором колонок-атрибутов. Наша матрица — **pivot** (позиции × классы), где
   колонки суть вторая ось данных (классы), а ячейка = множество вендоров.
   Структурно это разные таблицы.
2. У «позиции/стандарта» **нет собственных атрибутов**, которые заказчик хотел бы
   просматривать/править списком, кроме «какие вендоры в каком классе». Значит
   отдельный экран-каталог был бы **дублёром матрицы** — не делаем.
3. Пункт сайдбара «Каталог стандартов» уже ведёт на `/matrix`
   ([AppSidebar.tsx:36](../../../frontend/src/components/layout/AppSidebar.tsx#L36));
   матрица и есть каталог. Значит правильный ход — **перенести визуальный язык
   `ReviewItemsTable` на существующую матрицу**, не плодя экран.

## Цель

Пересобрать визуальный слой экрана матрицы под язык `ReviewItemsTable`, сохранив
pivot-механику (TanStack Table + динамические колонки-классы). Никакого backend,
миграций, новых маршрутов, нового API и инлайн-правки.

## Scope

### Входит

- **Sunken-шапка** матрицы: фон `bg-muted`/плотнее, uppercase-подписи колонок с
  трекингом (наш `text-caption`), две полосы (группы классов → листья), sticky-top.
- **Липкая первая колонка «Позиция»** при горизонтальном скролле (сейчас её нет):
  `position: sticky; left: 0` + фон + тень-край.
- **Чип-язык ячеек** в `renderCell`:
  - вендор — компактный чип (border + фон карточки);
  - действующее соглашение (`starred`) — mint-заливка + ★ mint (наш `--mint`/`chart-2`);
  - `ujin_integration` — приглушённая приписка `Ujin` (левый бордер-разделитель);
  - требование (`spec_text`) — warning-tan бейдж (наш `--warning`);
  - пусто — тихий `—` (`text-muted-foreground`, пониженная непрозрачность).
- **Строки-разделы** (`category_path`): плотнее, sticky-left, uppercase-подпись
  с трекингом; голова/уточнение позиции разнесены на две строки через `splitQualifier`.
- **Плотность**: сниженные паддинги ячеек, `tabular-nums` где числа, `hover`-подсветка
  строки и липкой ячейки.
- **Лёгкая шапка экрана**: заголовок «Каталог стандартов» + одно-строчный подзаголовок
  над тулбаром фильтров (сейчас экран без заголовка; крошка в `AppHeader` остаётся).
- **Обе темы** — только через семантические токены (не инверсия): работает
  автоматически, отдельного dark-кода не пишем.

### Не входит (осознанно)

- **Инлайн-правка ячеек** — отдельная фича уровня ТЗ §6: ячейка = множество вендоров
  (add/remove на пересечении), не скаляр как в образце. Экран остаётся read-only.
- **Backend / endpoint / миграции / схема БД** — ноль изменений.
- **Маршруты и навигация** — `/matrix` и метка сайдбара без изменений (метка уже честна).
- **Фильтры-контролы** (тип объекта / класс / поиск) — логику и URL-контракт не трогаем;
  допустима косметика обёртки, но поведение то же.
- **Пагинация** — механику не трогаем; допустима косметика.
- **Перенос токенов UDP 1-в-1** — UDP-имена (`surface-sunken`, `fg-tertiary`,
  `accent-soft`, `2xs`) НЕ вводим; используем наши semantic-токены.

## Дизайн (пофайлово)

### `frontend/src/screens/matrix/columns.tsx`

- `renderCell` — заменить `Badge variant="outline"` на чип-язык. Чипы — **экранные
  утилитарные классы прямо здесь**, НЕ новые DS-примитивы (золотое правило: в
  `components/ui/` только shadcn). Требование `spec_text` — оставить на `Badge
  variant="requirement"`, если его тон уже warning-tan; иначе выровнять к `--warning`.
- Колонка «Позиция» (`positionCol`) — рендер головы/уточнения двумя строками через
  `splitQualifier`.
- Прокинуть `meta.className` в дефы колонок: колонка «Позиция» получает sticky-классы
  (`sticky left-0 z-…` + фон), группы-старты — левый бордер.

### `frontend/src/screens/matrix/model.ts`

- `splitQualifier` живёт в `screens/vendors/model.ts`. Решение: **не тянуть импорт
  между экранами** — либо (а) продублировать крошечный чистый хелпер в
  `matrix/model.ts`, либо (б) поднять в общий модуль. Выбор — в плане; по умолчанию
  (а) как минимально связный (helper ~10 строк, чистый, покрыт тестом).
- Возможен хелпер разметки sticky/border-классов, чтобы `columns.tsx` и рендер
  строк-разделов в `MatrixScreen` не расходились.

### `frontend/src/screens/matrix/MatrixScreen.tsx`

- Обёртка таблицы: контейнер с `overflow-x-auto` (у shadcn `Table` он есть — проверить,
  не дублировать), sunken `TableHeader`, sticky-top шапка.
- Прокинуть `meta.className` в `TableHead`/`TableCell` (сейчас className не читается
  из дефа): `cn(base, header.column.columnDef.meta?.className)` и то же для ячеек.
- Строки-разделы: sticky-left ячейка (иначе при скролле подпись раздела «уедет»),
  тон `--section-bg` (в мокапе — чуть плотнее muted).
- Лёгкая шапка экрана (title + subtitle) над фильтр-`Card`.
- Плотность/hover — классами; фильтры и пагинатор структурно не трогаем.

## Ключевые решения

- **Стиль поверх структуры.** Фиксированный CSS-grid из `ReviewItemsTable` тут
  неприменим (число классов-колонок динамическое) — переносим только язык поверх
  TanStack-таблицы. Это записано, чтобы реализатор не начал переписывать pivot.
- **Чипы — экранные, не DS.** Специфичны для матрицы; в `components/ui/` не выносим.
- **Read-only.** Режим «просмотр»; никакого `fieldset`-гейта в этом срезе.
- **Токены — наши.** Соглашение = mint (`--mint`/`chart-2`), требование = `--warning`
  (tan). Обе темы бесплатно через semantic-токены.

## Тестирование

- **vitest:** существующие тесты матрицы должны остаться зелёными (структура строк/
  колонок и URL-контракт не меняются). Добавить точечный тест на `renderCell`:
  соглашение → есть ★/mint-класс; требование → текст `spec_text`; пустая ячейка → `—`.
- **a11y:** ссылки-вендоры остаются `<Link>` (не кнопки), focus-ring сохраняем;
  ★ несёт `aria-label` (как сейчас).
- **Проверка:** перед пушем полный `just ci` (prettier/tsc/vitest — фронтовый контур;
  `.py` не трогаем, но `just ci` гоняем целиком).

## Риски и ловушки

- **twMerge роняет кастомные размеры** (урок из where-allowed-edit-clipping): при
  композиции className не полагаться на дефолт shadcn, задавать явные размеры/тон.
- **Липкая колонка требует фона** на каждой sticky-ячейке (шапка/тело/разделы), иначе
  контент просвечивает при скролле; z-index: шапка > липкая колонка > обычные ячейки.
- **Строки-разделы** сейчас `colSpan={leafCount}` — при sticky-left нужно, чтобы
  подпись жила в липкой зоне, а не растягивалась под скролл.
- **`Badge variant="requirement"`** — свериться с его текущим тоном перед правкой,
  чтобы не ввести второй источник цвета требования.
