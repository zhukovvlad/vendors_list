# Bug reports — разборы пойманных багов

Папка для детективных историй: баг → как искали → корень → фикс → уроки.
Не дублирует devlog (там хроника задач) и TECH_DEBT (там осознанные долги) —
здесь именно **расследования**: что сломалось, почему это было неочевидно,
какие ложные следы съели время и какой урок уносим.

**Формат файла:** `YYYY-MM-DD-краткий-слаг.md`, внутри — секции
«Симптомы», «Расследование», «Корень», «Фикс», «Уроки». Пишем простым языком,
но без упрощения по существу: механизм бага должен быть объяснён так, чтобы
через год можно было понять и не наступить снова.
