# 2026-07-10 — Дашборд «Обзор» (начальный экран, фаза 1)

Ветка `feat/dashboard-overview`. Реализовано по методике subagent-driven-development
(план [docs/superpowers/plans/2026-07-10-dashboard-overview.md](../superpowers/plans/2026-07-10-dashboard-overview.md),
спека [.../specs/2026-07-10-dashboard-design.md](../superpowers/specs/2026-07-10-dashboard-design.md)):
11 задач (failing-first в каждом шаге), свежий субагент-исполнитель на каждую +
ревью между тасками, финальное whole-branch ревью перед PR.

Цель — начальный **read-only** экран приложения: три метрики (Позиции / Издания /
Вендоры), список открытых черновиков «в работе» и очередь «Требует внимания»
(похожие вендоры + залежавшиеся черновики). Сквозной срез БД → бэкенд → фронт.

## Что сделано

- **Вьюхи `dashboard_summary` + `dashboard_open_drafts`** (миграция `0004`,
  чистый SQL через `op.execute`, рабочий `downgrade`). `dashboard_summary` — одна
  строка скалярных агрегатов; `dashboard_open_drafts` — строка на открытый черновик
  с `last_touched_at`/`last_touched_by` (LATERAL по `listing` типа объекта, fallback
  на `release.created_at`/`author`). db-тесты — инвертированный TDD, дельта от базовой
  линии (вьюхи глобальны над засеянной базой).
- **`GET /dashboard`** — read-only эндпоинт (`Depends(read_conn)` + `require_user`,
  без `tx`) поверх вьюх. `is_stale` считается **в запросе** через
  `make_interval(days => :d)`, `:d` = `Settings.dashboard_stale_days`. Схемы
  `Dashboard{summary, drafts}`.
- **Детект похожих вендоров** (`app/services/dashboard.py`, прикладной слой, не БД):
  `normalize_vendor_name` (lower + срез общего хвоста «(Native)» + схлопывание
  пунктуации, БЕЗ срезания скобок с брендом-владельцем) + `count_merge_candidates`
  (коллизия нормы между разными бренд-ключами). Устойчив к таймауту: `SET LOCAL
  statement_timeout` + перехват `DBAPIError` → `None`, чтобы медленный детект не
  ронял весь дашборд.
- **Фронт-экран** `screens/dashboard/` — `DashboardScreen` (метрики/черновики/
  внимание, скелетоны загрузки, mobile-first адаптив) + чистые хелперы `model.ts`
  (`formatRelative` с русским склонением, `hasAttention`). Хук `useDashboard`
  (TanStack Query) + MSW-фикстура.
- **Роутинг**: дашборд — на `/`; матрица перенесена на `/matrix` (редирект-цель
  лоадера и `initialEntries` матричных тестов обновлены).
- **DS**: статус-токен `warning` (light `#9A6636` / dark `#BD9375` = `chart-3`) и
  компонент `skeleton` (на `bg-accent`).

## Ключевые решения

- **Позиции = снимок последнего `published`-релиза на тип объекта.** CTE
  `DISTINCT ON (building_type_id)` с детерминизмом `effective_date DESC NULLS LAST,
  frozen_at DESC NULLS LAST, id DESC` (страховка от равных дат) — считаем `count(DISTINCT
  position_id)`.
- **Вендоры по бренд-ключу** `coalesce(represents_id, id)`: суб-вендоры схлопнуты в
  владельца; «с соглашением» — через `vendor_starred(brand_id)`.
- **`is_stale` — в запросе, не во вьюхе.** Порог правится env (`DASHBOARD_STALE_DAYS`,
  default 14), не миграцией.
- **Детект дублей деградирует в `null`, не 500** — двойная защита (внутренний
  `except DBAPIError` в сервисе + внешний `except` в роутере). «Пропуск лучше шума».
- **Экран строго read-only:** «Новый стандарт» и шевроны рендерятся, но инертны
  (развилка O4) — их цели (редактор/списки) впереди.
- **Залежавшийся черновик — «мягкая подсказка, не приговор»:** показывается и в
  «Черновики в работе» (полный список открытых), и во «Требует внимания» — дубль
  намеренный (см. ниже: ловушка, пойманная ревью).

## Ловушки, пойманные ревью

- **Экран отфильтровывал залежавшиеся из «Черновики в работе».** Первая реализация
  Task 10 сделала `activeDrafts = drafts.filter(!is_stale)`, что противоречит спеке
  (§«Черновики в работе» = ВСЕ открытые; «Требует внимания» собирается отдельно
  через `drafts.filter(is_stale)`) и могло дать ложное «черновиков нет», когда все
  открытые — залежавшиеся. Откат к рендеру всех черновиков; тест усилен на
  `getAllByText(...).toHaveLength(2)` (дубль в двух панелях — это и есть контракт).
- **Баг в тест-данных плана.** db-тест детекта в плане брал пары `"ZzBrand"`/`"zz
  brand"` и ждал коллизию, но `normalize_vendor_name` сохраняет границы слов
  (`"zzbrand"` ≠ `"zz brand"`) → пара не схлопывается. Исправлено на однословные
  формы (`"zzbrand"`/`"yybrand"`), которые реально совпадают.
