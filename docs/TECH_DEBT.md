# Техдолг и отложенная работа

Живой реестр осознанно отложенных доработок: то, что **решено не делать сейчас**,
но нужно не потерять. Не баг-трекер и не бэклог фич — только уже принятые
компромиссы с указанием причины и где «болячка» проявится.

**Как вести:** добавляй пункт в нужную секцию с одной строкой «почему отложено».
Когда пункт закрыт — удаляй его (история останется в git и devlog). Источник
истины по деталям — спека/план соответствующей задачи (ссылки внизу секций).

## Дизайн-система (foundation)

Внесено при интеграции MR Design System (ветка `feat/design-system-integration`).
Foundation — только токены/темы/шрифты/реколор `button`; всё ниже сознательно
вне объёма. Источник: [спека](superpowers/specs/2026-07-08-design-system-integration-design.md)
(§«Вне объёма», §«Polish backlog») и [план](superpowers/plans/2026-07-09-design-system-integration.md).

### Шрифты
- **woff2-сжатие.** Самохостим сырые `.otf`/`.ttf` — тяжелее и без сжатия.
  Прогнать через woff2 (обычно −40–60% веса) при полишинге прода.
- **FOUT / `size-adjust`.** У Suisse и Inter (fallback) разные метрики → при
  загрузке шрифтов возможен сдвиг лейаута на первых кадрах. Лечится
  `size-adjust` в `@font-face` или `font-display: optional`.
- **Grtsk Giga без веса 400.** Самохостится в 100/300/500; дисплейный текст под
  `font-display` без явного веса подтянет ближайший (500 Medium). Для заголовков
  это ожидаемо — пункт-напоминание, чтобы никто не «чинил» фантом.
- **`font-synthesis: none` глобально.** Убирает синтетический жирный и в
  контентном тексте с пользовательской разметкой (`<strong>`/`<b>`). Для UI
  правильно; в контентных зонах с чужой разметкой включить синтез точечно.

### Компоненты
- **Focus-ring на `destructive`-кнопке.** Сейчас фиолетовый `--ring` на красной
  кнопке в фокусе — инородно. Дать destructive отдельный ring-цвет при полировке.
- **`--destructive-foreground`.** Определён в `:root`/`.dark`, но без маппинга в
  `@theme inline` и без потребителя (сплошная danger-кнопка использует литерал
  `text-white`). Добавить `--color-destructive-foreground` при появлении
  soft-danger компонентов — или убрать переменную до момента надобности.
- **Точные px-размеры кнопки.** Оставлены shadcn (`h-8`/`px-2.5`), не переведены
  в DS `11×24` — полиш под конкретные экраны.
- **`disabled` через `opacity`.** Оставлен shadcn-механизм (в тёмной поднят до
  `opacity-60`), не DS-реколор `border-strong`/`fg-subtle` — полиш.
- **Каталог компонентов.** Следующий срез (отдельный PR): `card` + `badge` +
  `table` — несущие для матрицы соответствия/светофора/статусов. Настоящие
  болячки реколора (ховеры меню, бейджи-статусы, заголовки таблиц) вылезут на них.

### Чарты
- **Категориальная палитра.** `--chart-4/5` сейчас `provisional`; хендофф
  санкционирует для графиков только три хью (violet/mint/tan). Финал палитры на
  5+ рядов — дизайн-решение, финалим с реальными сериями на дашборде проектов.

## Сид-импорт Excel (разовый)

Внесено при реализации разового сида вендор-листов (ветка `feat/excel-seed-import`).
Источник: [спека](superpowers/specs/2026-07-09-excel-seed-import-design.md) (§8, §9, §11, §16)
и [план](superpowers/plans/2026-07-09-excel-seed-import.md); финальное ревью ветки.

- **Предупреждение о рассинхроне сетки групп офиса (§11) не реализовано.**
  `reader.ClassColumn.group_name` вычисляется, но `loader`/`report` его не потребляют —
  «громкое предупреждение» при расхождении сетки групп с `segment_group` (§11, помечено
  как не-блокер) отложено. Пока метки классов офиса совпадают с `segment.name` дословно,
  сетка сходится по позиции колонки; реализовать при первом реальном расхождении шапки.
- **Смешанная ячейка «По согласованию…, <вендоры>» молча роняет вендоров (§8.4).**
  `match_requirement` для `Россия` — точное совпадение (смешанная ячейка уходит в
  vendor-list + `Требование:` в note), а для `По согласованию` — префикс, поэтому
  «По согласованию…, Bosch» матчится как ячейка-требование целиком и `Bosch` теряется
  без предупреждения. В 3 стартовых файлах все `По согласованию`-ячейки — whole-cell,
  потерь нет. Закрыть, если источник начнёт смешивать `По согласованию` с вендорами
  (увести префиксные маркеры в ту же mixed-ветку, что и `Россия`).
- **Отчёт (§16) без части справочных счётчиков.** Не выводятся: число одиночных `Ujin`,
  список имён в кавычках, извлечённые note-области, эвристика «похожих» имён вендоров.
  Все — информационные; на корректность данных не влияют.
- **Метка freeze — ISO-дата.** `label` = «к Стандартам, 2026-03-25» вместо
  дотированной «25.03.2026» из спеки §13. Косметика; freeze по умолчанию выключен.
- **Эффект таймаутов сида не покрыт автотестом.** `_apply_timeouts` ставит
  `statement_timeout`/`idle_in_transaction_session_timeout`; тест проверяет их
  ПРИМЕНЕНИЕ через `SHOW` (ловит ошибку единиц измерения), но реальный обрыв по
  таймауту не воспроизводится без медленного оператора (`pg_sleep`) — сознательно
  не тестируем. Эффект верифицируется инспекцией.
