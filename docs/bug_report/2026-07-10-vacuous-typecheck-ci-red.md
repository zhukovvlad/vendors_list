# CI красный, локально зелёный: тайпчек, который ничего не проверял

**Дата:** 2026-07-10 · **Ветка:** `feat/listing-matrix` (PR #12) · **Серьёзность:** красный CI + дыра в проверках
**Файлы:** `frontend/package.json`, `frontend/tsconfig.json`, `frontend/src/screens/matrix/MatrixScreen.tsx`
**Фикс-коммит:** `48f16f7`

## Симптомы

GitHub CI (job `frontend`) падал на сборке:

```
TS2339: Property 'offset' does not exist on type
'{ offset: number; ... } | {}'   (MatrixScreen.tsx:192, :206)
```

При этом локальный `just ci` — который прогоняет тот же `npm run typecheck` —
был полностью зелёным на том же коммите. Классическая «у меня работает».

## Расследование

Гипотезы отрабатывали по одной, с проверкой фактом:

1. **Версии расходятся?** Нет: `@tanstack/react-router` 1.170.17 и
   `typescript` 6.0.3 совпадали в `node_modules`, `package.json` и
   `package-lock.json`.
2. **Грязный `node_modules` / stale-кэш?** Нет: переустановка и свежая
   регенерация `schema.d.ts` — локальный тайпчек всё равно зелёный.
3. **А что вообще исполняется?** Вот оно. `typecheck` был `tsc --noEmit`, а
   корневой `tsconfig.json` — это `"files": []` плюс `references` на
   `tsconfig.app.json`/`tsconfig.node.json` (стандартная раскладка Vite).
   Без флага `-b` tsc **не обходит references**: список файлов пуст → проверено
   **ноль файлов** → «зелёный». Настоящий тайпчек жил только в
   `npm run build` (`tsc -b && vite build`) — а его локальный `just ci` не
   запускал, CI же запускал build отдельным шагом.

Проверка гипотезы одной командой: `npx tsc -b` локально — и обе ошибки
воспроизвелись ровно как в CI.

## Корень

Двухслойный:

1. **Реальная ошибка типов.** Колбэк-апдейтер поиска у TanStack Router типизирован
   как `(prev: Search | {}) => ...` — навигация возможна с любого маршрута, где
   search пустой. Обращение `p.offset` на ветке `{}` некорректно.
2. **Проверка-пустышка**, которая эту ошибку год могла бы прятать: тайпчек,
   проверяющий ноль файлов, опаснее отсутствующего — он создаёт ложную
   уверенность и «зелёный» бейдж.

## Фикс

- `p.offset` → `("offset" in p ? p.offset : 0)` в обоих хендлерах пагинации
  (сужение union по `in`).
- `"typecheck": "tsc --noEmit"` → `"tsc -b"` — теперь локальный `just ci`
  проверяет то же, что CI-сборка, и этот класс ошибок ловится до пуша.

## Уроки

- **Вакуумная проверка хуже отсутствующей.** Если проверка всегда зелёная —
  однажды спроси, а что она реально проверяет.
- **Расхождение «CI красный / локально зелёный» — это всегда вопрос «что
  именно исполняется в каждом из мест»**, а не повод дёргать версии наугад.
- В проектах с Vite-раскладкой tsconfig (`files: [] + references`) команда
  тайпчека обязана быть `tsc -b`. Плоский `tsc --noEmit` на таком корне — no-op.
