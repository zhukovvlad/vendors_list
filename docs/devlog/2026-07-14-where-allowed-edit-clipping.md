# 2026-07-14 — Клиппинг «Где разрешён» в edit + полоса v4

Багфикс + точечный полиш блока «Где разрешён» карточки вендора. Follow-up к
срезу `feat/vendor-card-edit-mode` (PR #23, уже влит) — отдельным мелким PR.

## Симптом

При входе в режим правки контент раскрытых секций аккордеона «Где разрешён»
обрезался снизу: вторая позиция видна наполовину, хвост зоны редактирования
недоступен.

## Корень

DS `AccordionContent` ([components/ui/accordion.tsx](../../frontend/src/components/ui/accordion.tsx))
несёт `overflow-hidden` на обёртке и **жёсткую** `h-(--radix-accordion-content-height)`
на внутреннем `div`. Radix замеряет эту CSS-переменную в момент открытия секции.
При входе в edit одновременно (а) все секции раскрываются принудительно (смена
`value`) и (б) контент ВЫРАСТАЕТ — чипы получают `×` и переносятся, появляются
`⊖` и «+ класс». Высота фиксируется ДО дорастания; `overflow-hidden` режет хвост.

## Фикс — вариант B

Кастомная контент-обёртка `WhereAllowedContent` на уровне экрана (DS-примитив не
форкаем — золотое правило; триггер блока и так кастомный). В edit-режиме рендерит
`AccordionPrimitive.Content` БЕЗ анимации, БЕЗ `overflow-hidden` и БЕЗ фикс-высоты
(height auto, overflow visible) — секции всё равно раскрыты разом, анимация не
нужна. В view-режиме поведение DS сохранено (контент статичен — баг не проявляется).

## Тем же патчем

1. **«+ стандарт» выше легенды.** По макету легенда закрывает блок последней:
   …секции → «+ стандарт» → легенда. Кнопку вынес в переиспользуемый
   `plusStandardEl` (пустое и непустое состояние).
2. **Чистые полосы в edit.** Сводка «N позиций · все классы» в полосе стандарта
   гейтится `!editMode` — в правке полоса растёт в чипы, сводка полуправдива и
   конкурирует с кебабом. В edit-полосе только шеврон + имя + кебаб `⋯`.
3. **Типографика v4** (замена v3: «уровни сливались» при разводке одним весом).
   Уровни разведены типографской сигнатурой: полоса стандарта → `text-caption`
   UPPERCASE `tracking-[0.09em]` (капс = «группировка, не контент»), счётчик →
   `text-caption` без капса; имя позиции → `text-[15px] tracking-tight`
   `text-foreground` (единственный крупный текст блока), скобочное уточнение →
   `text-[13px]`. Только семантические токены/утилиты, новых токенов нет.

## Тесты

6 смоук-тестов в [VendorCardScreen.test.tsx](../../frontend/src/screens/vendors/VendorCardScreen.test.tsx):
контент edit без `overflow-hidden`/анимации; view сохраняет их; «+ стандарт» выше
легенды (`compareDocumentPosition`); полоса без счётчика/«все классы» в edit;
v4 — полоса caption/uppercase, имя позиции 15px, размеры разные. Честную высоту в
jsdom не проверить (по ТЗ) — визуальная проверка вынесена в PR-чеклист.

Ловушка: в edit-режиме кебаб стандарта (`aria-label="действия стандарта Жилой
дом"`) тоже матчит `/Жилой дом/` → для триггера полосы селектор якорим `/^Жилой дом/`.

## Правки по ревью

**Тон (заказчик, по скрину view-режима):** имя раскрытого стандарта читалось ярко
(`group-data-open:text-foreground`) → тон muted в обоих состояниях (закрытый
`text-muted-foreground/70`, открытый `text-muted-foreground`). Чипы классов были
яркими и того же размера, что позиции: `badgeVariants.outline` = `text-foreground`
+ `font-medium`, а twMerge вдобавок **ронял** кастомный `text-caption` (спутав с
`text-foreground`) → чип инheritил ~14px. Явно `text-[11px] font-normal
text-muted-foreground` (arbitrary-размер twMerge не путает с цветом).

**Гонка раскрытия (CodeRabbit, Major):** кнопка «Редактировать» видна, как только
загрузился вендор, а `where-allowed` может ещё грузиться. Старый код сеял `expanded`
из `standards` на клике → при входе в edit до прихода дерева `expanded=[]`, и
контролируемый accordion оставался свёрнут. Решение — НЕ сеять, а **выводить**:
храним только явно свёрнутые пользователем секции (`collapsed`), раскрытые = «все
минус свёрнутые». «Раскрыто разом» становится производным от данных (пересчёт при
приходе стандартов), без seed-эффекта. Побочно снят и лог `react-hooks/set-state-in-effect`
(первый заход был через `useEffect`+`setState` — линтер справедливо забраковал).
Регресс-тест: never-resolving хендлер where-allowed → вход в edit → `qc.setQueryData`
имитирует приход дерева → секции авто-раскрыты (сетевую async-доставку MSW в
подвешенном хендлере в этом сетапе доставить до react-query не удаётся — кэш надёжнее).
