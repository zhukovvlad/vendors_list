# Фриз вкладки при смене класса: бесконечный цикл рендеров без единой ошибки

**Дата:** 2026-07-10 · **Ветка:** `feat/listing-matrix` (PR #12) · **Серьёзность:** полный фриз вкладки
**Файлы:** `frontend/src/screens/matrix/MatrixScreen.tsx`, `frontend/src/api/queries.ts`
**Стал горячим после:** `3b4dcd6` (фикс замечания ревью) · **Фикс-коммит:** `66c86b1`

## Симптомы

Самый неприятный набор из возможных — **ни одной улики на поверхности**:

- смена класса → вкладка намертво зависает;
- консоль **пустая** — ни ошибки, ни ворнинга;
- DevTools → Performance вечно висит на «loading trace» (запись не поднимается);
- в логе бэкенда запросы завершаются 200 — и затем минуты полной тишины;
- пользователь верно заметил: «до фиксов замечаний ревью такого не было».

## Расследование

1. **Backend отпал сразу:** прямые запросы к `/listings/matrix` с `segment_id`
   отвечали 200 за ~1.5с; в логах не было ни ошибок, ни шторма запросов.
2. **jsdom-тесты не воспроизводили ничего** — даже со «взрослой» формой ответа
   (одна суженная колонка, как отдаёт реальный бэкенд).
3. **Реальный Chromium (Playwright):** 17 сценариев смены класса — жилой, офис
   с групповыми шапками, с `q`, циклы туда-обратно — все зелёные. Провокации
   «взаимодействие в окно рефетча» — тоже. Головоломка: у пользователя виснет
   стабильно, у драйвера — никогда.
4. **Перечитывание улик** дало ключ: «Performance trace не поднимается» =
   запись содержит миллионы событий = **шторм рендеров**. «Запросы в логе
   завершаются, потом тишина» = сеть работает, но её колбэки **не исполняются**.
   Пустая консоль = ошибки не выбрасывается. Всё сходится на синхронном цикле,
   душащем event loop.

## Корень

Три безобидных поодиночке вещи, вместе — фриз:

1. `useReactTable({ data: matrix.data?.items ?? [] })`. При смене класса
   меняется queryKey → на время рефетча `matrix.data === undefined` → инлайновый
   `?? []` отдаёт таблице **новый массив на каждый рендер**. Нестабильная
   ссылка `data` — документированная ловушка бесконечного ре-рендера из FAQ
   TanStack Table (внутренние авто-сбросы таблицы реагируют на «новые данные»
   и планируют обновление состояния → новый рендер → снова новый `[]` → цикл).
2. До правок ревью это окно было холодным: `getRowModel()` вызывался только
   внутри перебора строк, а строк в пустом окне нет. Фикс замечания «O(n²) →
   Map» сделал вызов `getRowModel()` **безусловным на каждый рендер** — и окно
   стало горячим.
3. Цикл рендеров синхронный — он **не отпускает event loop**. Ответ рефетча
   давно пришёл, но его колбэк не может выполниться → `matrix.data` никогда не
   станет определённым → цикл не выходит сам. Отсюда «вечность» фриза, пустая
   консоль (React не кидает ошибку) и неподъёмный Performance-trace.

Почему не воспроизводилось у автоматического драйвера — честный ответ: не
знаем до конца. Поджиг цикла требует «лишнего» рендера внутри узкого окна
рефетча; реальный пользовательский ввод (фокус/ховер/скролл native select)
даёт его, синтетический `selectOption` — по-видимому, нет. Это зафиксировано
как ограничение, а не как объяснение.

## Фикс

Двумя слоями, оба — устранение корня, не симптома:

- **`EMPTY_ITEMS`** — стабильная модульная константа вместо инлайнового
  `?? []` (прямая рекомендация FAQ TanStack Table). Класс цикла «нестабильная
  ссылка data» устранён в принципе.
- **`placeholderData: keepPreviousData`** в `useMatrix` — окна пустых данных
  больше нет вовсе: при рефетче таблица держит предыдущие строки. Бонус к UX:
  пропало мигание пустой таблицей на ~1.5с латентности Neon при каждой смене
  фильтра.

Для проверки на стороне пользователя применялся некоммитный детектор шторма:
счётчик рендеров, пишущий в `document.title` — заголовок вкладки обновляется
синхронно и виден даже при полном фризе страницы. Приём стоит запомнить.

## Уроки

- **Инлайновый `?? []` (или `?? {}`) в props, чувствительных к ссылочной
  стабильности** (data таблиц, deps, контексты) — мина. Фолбэк должен быть
  модульной константой либо мемоизирован.
- **Замирание без ошибок в консоли — подозревай синхронный цикл** (рендеров или
  микротасок). Его сигнатуры: Performance-trace не поднимается, сетевые ответы
  приходят, но «не действуют», интерфейс мёртв.
- **Лог бэкенда — улика и для фронтовых багов:** «запросы завершаются и
  наступает тишина» отличает фриз фронта от зависшего запроса.
- **Заголовок вкладки (`document.title`) — канал диагностики, работающий при
  фризе**, когда консоль и DevTools уже недоступны.
- Правки по замечаниям ревью (тут — оптимизация O(n²)→Map) меняют поведение в
  углах, о которых замечание не говорило. Обе истории этого дня — регрессы от
  ревью-фиксов; вывод один: фикс замечания гоняется по сценариям так же, как
  любая фича.
